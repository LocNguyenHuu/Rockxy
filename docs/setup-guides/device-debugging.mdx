# Device Debugging

Rockxy supports device and simulator debugging through Developer Setup Hub, but each platform keeps its own trust, reachability, and proxy constraints.

## Device guides

Use the target-specific guide that matches where your app runs:

- [iOS Device](/setup-guides/ios-device)
- [iOS Simulator](/setup-guides/ios-simulator)
- [Android Device](/setup-guides/android-device)
- [Android Emulator](/setup-guides/android-emulator)
- [tvOS / watchOS](/setup-guides/tvos-watchos)
- [Vision Pro](/setup-guides/vision-pro)

## What these targets have in common

- They depend on certificate trust, not just proxy startup.
- Physical devices need a reachable host, not a localhost-only proxy.
- Simulators and emulators still require platform-specific trust and restart behavior.
