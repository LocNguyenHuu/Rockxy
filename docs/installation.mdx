# Installation

Rockxy is a native macOS application. Installation is straightforward, but HTTPS inspection only works once the certificate trust step is complete.

## System Requirements

| Requirement | Value |
|-------------|-------|
| Operating System | macOS 14.0 (Sonoma) or later |
| Processor | Apple Silicon or Intel |
| Disk Space | ~100 MB for the application |

## Download

Download the latest `.dmg` from [GitHub Releases](https://github.com/RockxyApp/Rockxy/releases).

Rockxy currently ships a universal macOS release package for both Apple Silicon and Intel Macs.

If future releases add architecture-specific downloads, the release notes will call that out explicitly. If you are unsure which Mac you have, choose the standard release listed on GitHub.

## Install

1. Open the downloaded `.dmg` file.
2. Drag **Rockxy.app** into the **Applications** folder.
3. Launch Rockxy from Applications or Spotlight.

## Trust the root certificate

<Frame caption="Install and trust the Rockxy root certificate.">
  <img src="/images/InstallCertificateRockxy.png" alt="Install and trust the Rockxy root certificate" />
</Frame>

<Warning>
HTTPS interception will not work until you install and trust the Rockxy root CA certificate. Without it, only plain HTTP traffic is captured.
</Warning>

Rockxy generates a unique local root CA on first launch. That certificate is used to mint per-host certificates when Rockxy decrypts HTTPS traffic for inspection.

### Install and trust it

1. Launch Rockxy.
2. Go to **Certificate > Install on This Mac...** in the menu bar.
3. Enter your macOS password if prompted. This adds and trusts the root CA through Rockxy's certificate flow.
4. Open **Keychain Access** (or follow the on-screen prompt). Locate the **Rockxy CA** certificate.
5. Double-click the certificate, expand **Trust**, and set **When using this certificate** to **Always Trust**.
6. Close the dialog and enter your password to confirm.

After you trust the certificate, restart any app that may have cached a failed TLS session. Some clients, such as Firefox or Java-based tools, keep their own trust path and still need app-specific setup. Use the [Setup Guides](/setup-guides/browsers-and-api-clients) when that happens.

For the full trust model, see [Certificates and Trust](/getting-started/certificates-and-trust).

## Verify your install

You should now be able to:

1. click **Start** in Rockxy,
2. load any HTTPS page or API request through the proxy,
3. open the request in Rockxy,
4. see readable headers and body instead of only a CONNECT tunnel.

If you cannot, continue with [First Capture](/getting-started/first-capture) or [Capture Issues](/troubleshooting/capture-issues).

## Data Storage Locations

Rockxy stores local data under the active app identity for the build you are running. In other words, the exact folder name and preferences plist follow the current bundle configuration rather than one fixed historical identifier.

| Data | Storage Method | Location |
|------|---------------|----------|
| User preferences | UserDefaults | `~/Library/Preferences/<Rockxy bundle identifier>.plist` |
| Root CA private key | macOS Keychain | Keychain Access > System keychain |
| Saved traffic and logs | SQLite | `~/Library/Application Support/<Rockxy app support directory>/rockxy.sqlite3` |
| Large response bodies | Disk files | `~/Library/Application Support/<Rockxy app support directory>/bodies/` |
| Traffic rules | JSON file | `~/Library/Application Support/<Rockxy app support directory>/rules.json` |

For current builds, Rockxy derives these names from its bundle identity at runtime. If you want to confirm the exact values on your Mac, open **Settings → Privacy** in the app, where Rockxy shows the active local storage paths directly.

## Build from Source

Requires Xcode 16+. Dependencies are managed through the Xcode project.

```bash
git clone https://github.com/RockxyApp/Rockxy.git
cd Rockxy
xcodebuild -project Rockxy.xcodeproj -scheme Rockxy -configuration Debug build
```

To run tests:

```bash
xcodebuild -project Rockxy.xcodeproj -scheme Rockxy test
```

## Uninstallation

<Warning>
Uninstalling Rockxy removes the application but does not automatically delete your saved sessions, rules, or the root CA certificate. Follow the steps below to remove everything.
</Warning>

1. Quit Rockxy if it is running.
2. Delete `Rockxy.app` from the Applications folder.
3. Remove the matching application-data directory and preferences plist for the build you installed. Typical current layouts are:
   ```bash
   rm -rf ~/Library/Application\ Support/com.amunx.rockxy
   rm -rf ~/Library/Application\ Support/com.amunx.rockxy.community
   rm -f ~/Library/Preferences/com.amunx.rockxy.plist
   rm -f ~/Library/Preferences/com.amunx.rockxy.community.plist
   ```
4. Open **Keychain Access**, find the **Rockxy CA** certificate, and delete it.

## Troubleshooting

### HTTPS traffic is not being captured

Verify the root CA certificate is installed and trusted. Open **Keychain Access**, search for "Rockxy CA", and confirm it shows "This certificate is marked as trusted for all users." See [Trust the root certificate](#trust-the-root-certificate) above.

### Proxy does not capture traffic from a specific app or runtime

Some apps ignore the macOS system proxy or require their own trust-store changes. Start with:

- [Browsers and API Clients](/setup-guides/browsers-and-api-clients)
- [Terminal Runtimes and Frameworks](/setup-guides/terminal-runtimes-and-frameworks)
- [Device Setup Issues](/troubleshooting/device-setup-issues)

### High memory usage with many captured requests

Rockxy holds up to 50,000 transactions in memory. Older transactions are evicted from the active in-memory buffer as needed. If memory usage is still high, clear the current session from **Flow > Clear Session**.

## Claude setup

If you want to use Rockxy with Claude, open **Settings → MCP** after installation and turn on **Enable MCP Server**.

Rockxy ships with a bundled local MCP bridge named `rockxy-mcp`. Both Claude CLI and Claude Desktop can use it, but they each keep their own MCP configuration.

See [MCP Integration](/features/mcp) for the full end-user guide, including:

- Claude CLI setup with `claude mcp add`
- Claude Desktop setup with `claude_desktop_config.json`
- troubleshooting for local MCP vs. remote Connectors
